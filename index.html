Detecting Non-Optimal Decisions of Embodied Agents via Diversity-Guided Metamorphic Testing

Paper Code

Abstract

As embodied agents advance toward real-world deployment, ensuring optimal decisions becomes critical for resource-constrained applications. Current evaluation methods focus solely on functional correctness, overlooking the non-functional optimality of generated plans. This gap can lead to significant performance degradation and resource waste. We identify and formalize the problem of Non-optimal Decisions (NoDs), where agents complete tasks successfully but inefficiently. We present a systematic framework for detecting NoDs in embodied agent task planning via diversity-guided metamorphic testing. Our key insight is that optimal planners should exhibit invariant behavioral properties under specific transformations. We design four novel metamorphic relations capturing fundamental optimality properties: position detour suboptimality, action optimality completeness, condition refinement monotonicity, and scene invariance. To maximize detection efficiency, we introduce a diversity-guided filter that actively selects test cases exploring different violation categories, avoiding redundant evaluations while ensuring comprehensive diversity coverage. Extensive experiments on the AI2-THOR simulator with four state-of-the-art planning models demonstrate our framework achieves violation detection rates ranging from 18.7% to 67.2% across different task complexities, with our diversity-guided filter enhancing detection efficiency by 4.3% on average, significantly outperforming four baseline methods by 40.7% in average detection rates.

Method

Our framework consists of three main components: metamorphic relations design, diversity-guided filtering, and violation detection. The overall architecture is shown below.

Metamorphic Relations

  • Position Detour Suboptimality: Detects inefficient path planning through detour analysis
  • Action Optimality Completeness: Identifies suboptimal action sequences and alternatives
  • Condition Refinement Monotonicity: Evaluates sensitivity to environmental condition changes
  • Scene Invariance Property: Tests robustness to task-irrelevant scene modifications

Diversity-Guided Filter

  • Position Filter: Maximizes perpendicular distance to original trajectory
  • Action Filter: Leverages LLMs for semantic diversity selection
  • Condition Filter: Maximizes refinement divergence from original conditions
  • Scene Filter: Minimizes scene similarity while maintaining task-irrelevance
